# `roadmap.md` 设计说明

语言：简体中文 | [English](ROADMAP.md)

## 1、该文件的作用

定义项目阶段目标与里程碑，回答“接下来做什么”。

## 2、设计理念

- 面向阶段，而非面向单次任务
- 可追踪里程碑完成度
- 与 `tasks.md` 解耦：路线定义目标，任务负责执行

## 3、内容或者子文件的编写风格/格式

- 按“阶段（Phase）→ 里程碑（Milestone）→ 验收结果（Outcome）”组织
- 每个阶段使用统一标题层级（建议二级标题）
- 每个里程碑建议包含：目标、时间窗口、风险与依赖
- 与 `tasks.md` 保持引用关系：任务需标注所属阶段
- 每个阶段建议固定字段：`目标`、`里程碑`、`风险`、`依赖`
- 里程碑条目建议使用勾选格式，状态一眼可见
- 时间建议统一 `YYYY-MM-DD` 或 `YYYY-QN`，避免混用
- `roadmap.md` 中每条里程碑/任务都必须包含唯一 ID
- 推荐 ID 格式：`ROADMAP-0001`、`ROADMAP-0002` ...
- ID 一旦发布禁止修改（可改标题和状态，不可改 ID）
- `designs/` 与 `reports/` 必须引用该 ID 以支持反向检索
