# `.projitive` 设计说明

语言：简体中文 | [English](PROJITIVE.md)

## 1、该文件的作用

`.projitive` 是治理定位锚点，用于声明“该目录是一个 Projitive 治理规划目录”。

## 2、设计理念

- 去路径耦合：不再依赖固定路径
- 就地治理：标记放在哪，治理根目录就在哪
- 可迁移：项目迁移目录结构时，治理能力不受影响

## 3、内容或者子文件的编写风格/格式

- 文件格式建议二选一：
	- 空文件（推荐，最稳定）
	- 极简键值（仅在需要扩展元数据时）
- 若使用键值格式，建议仅允许以下字段：
	- `version`：规则版本（如 `1`）
	- `name`：治理目录名（可选）
	- `owner`：默认治理负责人（可选）
- 一个治理根目录只放一个 `.projitive`，禁止在子目录重复放置
- 工具实现需支持“从任意子路径向上查找最近 `.projitive`”
- 找不到标记时需返回明确错误，禁止静默 fallback 到固定目录

建议示例（键值模式）：

```yaml
version: 1
name: expense-governance
owner: team-platform
```

## 4、内建治理检查建议（模块内）

- 每个治理根目录只应存在一个 `.projitive` 标记
- 若存在嵌套重复标记，应告警为歧义风险
- 向上查找最近标记的行为应保持确定性
- 缺失标记时必须显式报错（禁止静默 fallback）
